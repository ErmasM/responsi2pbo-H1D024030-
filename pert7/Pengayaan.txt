Pertanyaan Refleksi
Setelah menyelesaikan kode, jawab pertanyaan berikut:

1. Apa perbedaan antara interface dengan class biasa?
2. Mengapa interface tidak memiliki implementasi method di dalamnya?
3. Apa tujuan penggunaan keyword "implements" pada sebuah class?
4. Apakah sebuah class dapat mengimplementasikan lebih dari satu interface? Jelaskan dan berikan contoh sederhana.
5. Jika sebuah class mengimplementasikan interface, apa yang terjadi jika class tersebut tidak mengimplementasikan semua method dari interface tersebut?

Jawab:

1. Interface berbeda dengan class biasa karena interface hanya berisi deklarasi method tanpa implementasi, sedangkan class biasa bisa langsung berisi atribut, method lengkap, dan bisa dibuat objeknya. Interface lebih seperti kontrak yang harus dipatuhi oleh class yang menggunakannya, sementara class biasa sudah memiliki perilaku yang jelas dan bisa berdiri sendiri tanpa harus di-implement oleh class lain.

2. Interface tidak memiliki implementasi method karena tujuannya memang untuk menyediakan kerangka umum yang harus dimiliki oleh class yang menggunakannya. Dengan tidak adanya isi method, interface memberi kebebasan penuh kepada setiap class untuk menentukan sendiri bagaimana cara kerja method tersebut. Selain itu, sifat interface memang dirancang sebagai kontrak, jadi isinya hanya aturan, bukan cara kerja.

3. Keyword implements digunakan oleh sebuah class untuk menyatakan bahwa class tersebut bersedia mengikuti semua aturan dan method yang ada di dalam interface. Dengan memakai implements, class tersebut wajib menyediakan isi method yang sudah dideklarasikan di interface, sehingga memastikan bahwa class tersebut memenuhi standar atau perilaku tertentu sesuai kontrak yang telah ditentukan.

4. Sebuah class bisa mengimplementasikan lebih dari satu interface, dan ini salah satu kelebihan besar dari interface dalam Java. Hal ini memungkinkan sebuah class memiliki banyak kemampuan dari berbagai interface sekaligus, meskipun Java tidak mendukung multiple inheritance pada class. Misalnya:
class Burung implements BisaTerbang, BisaBersuara { ... }
Pada contoh ini, kelas Burung bisa memiliki perilaku terbang dan bersuara sekaligus karena ia mengimplementasikan dua interface berbeda.
 
5. Jika sebuah class mengimplementasikan interface tetapi tidak mengisi semua method yang ada di interface tersebut, maka class itu akan dianggap tidak lengkap dan menyebabkan error. Satu-satunya pengecualian adalah jika class tersebut dideklarasikan sebagai abstract, karena class abstract memang boleh tidak menyediakan implementasi. Tapi jika class-nya bukan abstract, Java akan memaksa programmer untuk mengisi seluruh method agar kontrak interface terpenuhi.
